Angular Routing and Using API Services

Angular Routing

We need to understand a little about how angular displays page's before we understand how routing works.

Angular is really a 'single page application'. That means the application is showing one page but the different views take turns appearing on Index.html page.

In this sample application, the Index.html file is a standard html file with two main tags: 'head' and 'body'. It uses standard boiler plate html, but within the body tag, there is an Angular custom html tag: 'app-root'. This custom tag is also the selector of the component we would like to appear in the body of our Index.html page. in this sample application, I have chosen to show the app.comonent in the Index.html body tag.

As you can see in app.comonent.ts file, the 'selector' is the same as the custom custom html tag in the Index.html file 'app-root'. And that is how we see the app.component.html file in our application.

In the image below, we can see the app.comonet.html file. You will notice the standard type of nav bar. But in the div tag just below the navigation section, there is the router-outlet. In other words, what ever the router selects, will appear in this section.

Now we know how the output of th router appears on the page, let's take a look at how the routing actually works.

Routing

Before I jump too far into Routing, I should mention app.modules. As far as I can tell, an angular app can have many different modules and many come standard with angular. But in this demo application, I have chosen to be simple and use only one module 'app.module.ts' which lives in the src/app folder.

If you will notice, there is an import statement for RouterModule. This is a necessity. Now we have to make this import available for all angular components in this application that will need to use routing. This is accomplished by adding RouterModule to the 'imports' portion after the @NGModule decorator.

Within the 'forRoot()' method of RouterModule we define the the path of the comonent to be router as the first parameter. The second parameter is the name of the component to be routed.

Finally we use a directive called 'routerLink' in the anchor tag as shown in 'Figure-3' above. And 'Presto' you got angular routing.

In individual components, it is possible to import 'Router' and use the navigate() method to router programmatically as well.
